# `requirements/` — normative specs referenced by multiple plans

Cross-cutting specifications and their illustrative artefacts (SVGs, diagrams,
worked examples). Live here when referenced by more than one plan; per-plan
visuals live next to the plan in `doing/` / `waiting-acceptance/` / `done/`.

Naming: kebab-case only; versioned images use `-v{N}-{label}` suffix
(`layout-rules-v2-curved.png`). No spaces in filenames.
